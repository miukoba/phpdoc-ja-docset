AMQPChannel::setPrefetchCount

(PECL amqp >= Unknown)

AMQPChannel::setPrefetchCount — ブローカからプリフェッチするメッセージの数を設定する

説明

public void AMQPChannel::setPrefetchCount ( int $count )

AMQPQueue::consume() あるいは AMQPQueue::get() メソッドのコールの間にブローカからプリフェッチするメッセージの数を指定します。 このメソッドをコールすると、自動的にプリフェッチウィンドウサイズを 0 にします。 つまりプリフェッチウィンドウサイズの設定を無視するようになるということです。

AMQPQueue::consume() あるいは AMQPQueue::get() を AMQP_AUTOACK フラグつきでコールした場合は、 この設定は無視されます。

パラメータ

count

プリフェッチするメッセージ数。

エラー / 例外

ブローカとの接続が途切れた場合に AMQPConnectionException をスローします。

返り値

成功した場合に TRUE を、失敗した場合に FALSE を返します。
