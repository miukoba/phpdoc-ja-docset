DOMText::__construct

(PHP 5)

DOMText::__construct — 新しい DOMText オブジェクトを作成する

説明

public DOMText::__construct ([ string $value ] )

新しい DOMText オブジェクトを作成します。

パラメータ

value

text ノードの値。指定しなかった場合は空の text ノードが作成されます。

例

例1 新しい DOMText を作成する

<?php

$dom = new DOMDocument('1.0', 'iso-8859-1');
$element = $dom->appendChild(new DOMElement('root'));
$text = $element->appendChild(new DOMText('root value'));
echo $dom->saveXML(); /* <?xml version="1.0" encoding="iso-8859-1"?><root>root value</root> */

?>

Thomas 27-Apr-2012 10:54
I experinced that DOMText, given a integer as only value to the constructor failed to return valid data. I had to typecast the input to a string first:

$intValue = 3252; // Post/Zip-code
$some_element->appendChild( new DOMText( (string) $intValue) );

When not typecasting using "(string)" the XML output from $root->saveXML() would not validate.
