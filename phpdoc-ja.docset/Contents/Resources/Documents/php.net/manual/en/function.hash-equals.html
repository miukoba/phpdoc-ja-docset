hash_equals

(PHP 5 >= 5.6.0)

hash_equals — Timing attack safe string comparison

説明

bool hash_equals ( string $known_string , string $user_string )

Compares two strings using the same time whether they're equal or not.

This function should be used to mitigate timing attacks; for instance, when testing crypt() password hashes.

パラメータ

known_string

The string of known length to compare against

user_string

The user-supplied string

返り値

Returns TRUE when the two strings are equal, FALSE otherwise.

エラー / 例外

Emits an E_WARNING message when either of the supplied parameters is not a string.

例

例1 example

<?php
$expected  = crypt('12345', '$2a$07$usesomesillystringforsalt$');
$correct   = crypt('12345', '$2a$07$usesomesillystringforsalt$');
$incorrect = crypt('apple',  '$2a$07$usesomesillystringforsalt$');

var_dump(hash_equals($expected, $correct));
var_dump(hash_equals($expected, $incorrect));
?>

上の例の出力は以下となります。

bool(true)
bool(false)

注意

注意:

Both arguments must be of the same length to be compared successfully. When arguments of differing length are supplied, FALSE is returned.

注意:

It is important to provide the user-supplied string as the second parameter, rather than the first.
